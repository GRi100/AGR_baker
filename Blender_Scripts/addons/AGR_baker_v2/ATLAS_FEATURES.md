# Atlas Features - AGR Baker v2

## Обзор

В аддон AGR Baker v2 добавлен полноценный функционал создания текстурных атласов из запеченных наборов текстур.

## Основные возможности

### 1. Автоматическое определение типа атласа

- **HIGH Atlas**: Создается по умолчанию, использует комбинированные текстуры:
  - `T_AtlasName_DO.png` - Diffuse + Opacity (RGBA)
  - `T_AtlasName_ERM.png` - Emission + Roughness + Metallic
  - `T_AtlasName_N.png` - Normal map

- **LOW Atlas**: Создается для объектов с специальным именованием (SM_Address_Main/Flora/Ground/GroundEl):
  - `T_Address_Type_d.png` - Diffuse
  - `T_Address_Type_r.png` - Roughness
  - `T_Address_Type_m.png` - Metallic
  - `T_Address_Type_o.png` - Opacity
  - `T_Address_Type_n.png` - Normal

### 2. Интеграция в список сетов

- Атласы отображаются в списке текстурных сетов с префиксом `A_`
- Визуально отличаются иконкой IMAGE_PLANE
- Показывают тип атласа (HIGH/LOW) и разрешение
- Хранятся в папках `A_AtlasName` внутри AGR_BAKE

### 3. Упаковка текстур

- Использует алгоритм Guillotine для оптимальной упаковки
- Сохраняет оригинальное разрешение каждой текстуры
- Автоматически масштабирует при необходимости (с использованием scipy если доступна)
- Проверяет возможность упаковки перед созданием атласа

### 4. Сохранение метаданных

Вся информация о раскладке атласа сохраняется в `atlas_mapping.json`:

```json
{
  "atlas_name": "A_ObjectName",
  "atlas_type": "HIGH",
  "atlas_size": 2048,
  "created_atlases": {
    "DIFFUSE_OPACITY": "path/to/texture.png",
    "ERM": "path/to/texture.png",
    "NORMAL": "path/to/texture.png"
  },
  "layout": [
    {
      "set_name": "S_MaterialName",
      "material_name": "MaterialName",
      "x": 0,
      "y": 0,
      "width": 1024,
      "height": 1024,
      "u_min": 0.0,
      "v_min": 0.0,
      "u_max": 0.5,
      "v_max": 0.5
    }
  ]
}
```

### 5. Автоматическое создание материала

- Создается материал с именем `M_AtlasName` или `M_Address_Type_1` для LOW атласов
- Автоматически подключаются все текстуры атласа
- Поддержка OpenGL и DirectX normal maps
- Правильная настройка цветовых пространств (sRGB для диффузных, Non-Color для остальных)

## Использование

### Создание атласа через UI

1. Откройте панель "Texture Sets" в AGR Baker
2. Выберите нужные текстурные сеты (галочки слева)
3. В секции "Atlas Operations":
   - Выберите размер атласа (512-8192px)
   - Выберите тип normal map (OpenGL/DirectX)
4. Нажмите одну из кнопок:
   - **Auto Atlas** - автоматически определяет тип на основе активного объекта
   - **HIGH Atlas (DO/ERM/N)** - принудительно создает HIGH атлас
   - **LOW Atlas (d/r/m/o/n)** - принудительно создает LOW атлас

### Именование объектов для LOW атласов

Для автоматического создания LOW атласов объект должен иметь имя формата:
- `SM_Address_Main` - основной объект
- `SM_Address_Flora` - растительность
- `SM_Address_Ground` - земля
- `SM_Address_GroundEl` - элементы земли

Где `Address` - уникальный идентификатор объекта.

### Пример workflow

1. Запеките текстуры для нескольких материалов
2. Обновите список сетов (Refresh Sets)
3. Выберите сеты для объединения в атлас
4. Выберите активным объект с нужным именованием (для LOW) или любой (для HIGH)
5. Создайте атлас
6. Атлас появится в списке с префиксом A_

## Технические детали

### Структура файлов

```
AGR_BAKE/
├── S_Material1_baked/          # Обычный текстурный сет
│   ├── T_Material1_Diffuse.png
│   ├── T_Material1_Normal.png
│   └── ...
├── S_Material2_baked/
│   └── ...
└── A_ObjectName/               # Атлас
    ├── T_ObjectName_DO.png     # HIGH naming
    ├── T_ObjectName_ERM.png
    ├── T_ObjectName_N.png
    └── atlas_mapping.json
```

или для LOW:

```
AGR_BAKE/
└── A_Address_Main/
    ├── T_Address_Main_d.png    # LOW naming
    ├── T_Address_Main_r.png
    ├── T_Address_Main_m.png
    ├── T_Address_Main_o.png
    ├── T_Address_Main_n.png
    └── atlas_mapping.json
```

### Свойства PropertyGroup

Добавлены новые свойства в `AGR_TextureSet`:
- `is_atlas` - флаг, что это атлас
- `atlas_type` - тип атласа (HIGH/LOW)
- `object_name` - имя объекта для которого создан атлас

### Операторы

- `agr.create_atlas` - создание атласа из выбранных сетов

### Зависимости

- **numpy** - обязательно для работы с пиксельными данными
- **scipy** (опционально) - для качественного масштабирования текстур
- **PIL/Pillow** (опционально) - для дополнительных операций с изображениями

## Улучшения по сравнению со старой версией

1. ✅ Полная интеграция в новую архитектуру аддона
2. ✅ Автоматическое определение типа атласа на основе активного объекта
3. ✅ Сохранение метаданных в JSON для возможности разборки атласа
4. ✅ Поддержка обоих схем именования (HIGH и LOW)
5. ✅ Визуальное отличие атласов в UI
6. ✅ Правильное именование текстурных нодов в материале
7. ✅ Улучшенная упаковка с проверкой возможности размещения

## Известные ограничения

- Атласы не могут быть включены в другие атласы (is_atlas сеты игнорируются при выборе)
- Разборка атласа пока не реализована (но данные сохранены в JSON)
- UV-коррекция объектов не реализована в текущей версии
